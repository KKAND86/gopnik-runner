- **Роль**: я — разработчик, пользователь — руководитель проекта. Обосновываю решения, согласовываю шаги, даю советы.

### Принятые решения (согласовано)
| Параметр | Значение |
|---|---|
| Платформа | Telegram Mini App (HTML5) |
| Движок | Phaser 3 (CDN ESM) |

### Что сделано
- Создана структура проекта `gopnik-runner/`
- Отказ от Vite — переход на ES modules + importmap + static server (Node.js 18 конфликты со сборщиком)
- Реализован Core Loop: игрок, прыжки, гопники, монетки, Game Over
- **Программный pixel-art**: все спрайты генерируются через Phaser Graphics API (16×16 → upscale)
  - Игрок (куртка, джинсы), Гопник (Adidas, тапочки), Монетка (₽), Бустеры (кефир, Nokia, мама)
- Интеграция Telegram Mini App API
- Dev-сервер (`npm run dev` → localhost:5173) и сборка (`npm run build` → dist/)

## 2026-05-13 (продолжение — отладка и итерации)

### Технические проблемы и решения
- **Dev-сервер**: процессы node падают в headless-окружении. Используем `Start-Process` для запуска в background. Порт 5173 часто занят — приходится убивать старые процессы.
- **Кэш браузера**: при каждом изменении нужно добавлять `?v=N` к `main.js` URL, иначе браузер грузит старую версию. Добавили `Cache-Control: no-store` в сервер.
- **Phaser CDN**: `cdn.jsdelivr.net` не отвечал — скачали `phaser.min.js` локально в `lib/` (1.1MB).
- **Загрузка**: `telegram.org/js/telegram-web-app.js` таймаутил 30+ сек. Убрали из HTML, загружаем динамически только если URL содержит `t.me`.

### Изменение концепции механики
Изначально гопники были препятствиями на пути (перепрыгивать). После тестирования переделали:
- **Гопники** — гонятся СЗАДИ (появляются за левым краем, догоняют)
- **Препятствия** — на пути (лужи, ящики, бочки) — их перепрыгиваем
- **Game Over** — либо гопник догнал, либо уперся в препятствие

### Проблемы с hitbox'ами и физикой
- `setOrigin(0.5, 1)` + `setOffset` ломали физику — игрок падал сквозь землю. Перешли на стандартный origin (0.5, 0.5).
- Прямоугольник земли 1280px обрывался — игрок убегал в пустоту. tileSprite `groundTile` + `grassTile` решает это.
- Ручная AABB коллизия давала ложные срабатывания. Добавили прощение (forgiveness): hitbox игрока -8px, hitbox препятствия -6px. Проверка: если низ игрока выше верха препятствия → перепрыгнул.

### Новые системы (добавлены в процессе отладки)
- **Уровни**: каждые 500м — "УРОВЕНЬ N", вспышка текста
- **Буст скорости от монеток**: +15 к скорости за каждую монетку (макс 600)
- **Индикатор скорости**: "XX км/ч" в правом верхнем углу
- **Индикатор опасности**: "⚠️ ГОПНИКИ СЗАДИ! Xм" когда гопник < 400px
- **Гопник**: спавн через 8 сек, на 800-1200px позади, начальная скорость 270 (меньше игрока)

### Итеративная отладка
Пользователь (PM) присылал скриншоты ошибок из Chrome DevTools. Исправляли по одной проблеме за раз:
1. `ERR_CONNECTION_REFUSED` — сервер не запущен
2. `Cannot read properties of undefined (reading 'graphics')` — `game.make.graphics` → `this.make.graphics` в сцене
3. `e[i] is not a function` — `Group.add()` конфликтовал с `classType`. Перешли на массивы + ручная AABB.
4. Чёрный экран — Phaser CDN не грузился → локальный файл
5. Пол обрывается → tileSprite
6. Гопник догоняет сразу → отдалил спавн, замедлил ускорение

### Уроки
- **Не использовать `setOrigin`/`setOffset` с Phaser физикой** — ломает коллайдеры. Стандартный origin (0.5, 0.5) + правильная позиция Y.
- **tileSprite > прямоугольник** для бесконечных фонов. Прямоугольник 50000px ломает WebGL.
- **Кэш браузера** — враг разработчика. Всегда добавлять версию к JS/CSS.
- **CDN не надёжен** для локальной разработки. Скачивать библиотеки локально.
- **Гопник должен быть СЗАДИ** — это напряжение, а не прыжки через гопников.
