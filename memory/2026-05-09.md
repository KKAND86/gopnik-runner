## Lessons
- Unity 6 (6000.x) changed package structure: `com.unity.ugui` 2.0.0 does NOT expose `UnityEngine.UI` assembly the same way as before
- Using `OnGUI()` avoids all UI assembly reference problems entirely
- Runtime object creation (Awake scripts) is more reliable than Editor scripts for quick prototyping
- Never use `renderer.material` in Editor scripts — use `sharedMaterial` to avoid material leaks
- Scene file corruption from broken script references can be fixed by creating a clean new scene
