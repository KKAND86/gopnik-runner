### Found pre-trained models

- Plus `.pkl` sklearn models trained on earlier numpy/sklearn versions

### Numpy/Sklearn Version Hell
- `.pkl` models saved with numpy 2.x + sklearn 1.8.x (outside Docker)
- Docker had numpy 1.26.3 + sklearn 1.4.0 → pickle load failed
- Tried upgrading to numpy 2.0.0 → broke OpenCV/TensorFlow (compiled against numpy 1.x)
- **Fix:** reverted to numpy 1.26.3 + sklearn 1.4.0, deleted old `.pkl`, regenerated new `.pkl` models from inside Docker container

### Feature Dimension Mismatch
- `_extract_sklearn_cv_features()` produces **37** features
- Original model generator used **20** features → `ValueError: X has 37 features, but StandardScaler is expecting 20`
- **Fix:** updated `generate_models.py` to auto-count features by calling actual extraction methods on dummy data before training

### MODELS_DIR Path Bug
- `BASE_DIR = Path(__file__).parent.parent.parent.parent` → `/` in Docker
- `MODELS_DIR = BASE_DIR / "models"` → `/models/` (doesn't exist)
- Actual models at `/app/models/`
- **Fix:** added Docker fallback path detection in `ml_inference.py`

### Priority Switch: TF → First
Original code prioritized sklearn over TF:
```python
if self._sklearn_cv:  # synthetic → always 0.3
if self.cv_model:     # real TF → never reached
```
**Fix:** swapped order so TF `.keras` models run first:
```python
if self.cv_model and self._tf_available:  # real trained weights
if self._sklearn_cv:  # fallback
```
