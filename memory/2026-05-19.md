
# 2026-05-19 — Gopnik-Runner Game Testing & Bug Fixes

## Session Summary
User asked to launch and test the gopnik-runner game for functionality and visuals. Found and fixed multiple bugs during testing.

## Bugs Discovered & Fixed

### 1. Express server wildcard route crash
- **File**: `gopnik-runner/server.js`
- **Issue**: `app.get('*', ...)` throws `PathError: Missing parameter name` in newer Express/path-to-regexp
- **Fix**: Changed to `app.get(/.*/, ...)` for SPA fallback

### 2. Phaser texture `sourceSize` null crash in OutroScene
- **File**: `gopnik-runner/src/scenes/OutroScene.js`
- **Issue**: OutroScene tried to play `gopnik_run` animation but textures/animations weren't initialized (only created in IntroScene)
- **Fix**: Added `generateAllSprites(this)` and `createAnimations(this)` to OutroScene.create()

### 3. Sprite texture recreation destroying animation frames
- **File**: `gopnik-runner/src/utils/SpriteGenerator.js`
- **Issue**: `renderPixels()` always did `textures.remove(key)` then recreate, which destroyed textures referenced by existing animations when scenes reloaded
- **Fix**: Added early return if texture already exists: `if (scene.textures.exists(key)) return;`

### 4. `addImage` vs `addCanvas` for texture aliasing
- **File**: `gopnik-runner/src/utils/SpriteGenerator.js`
- **Issue**: `textures.addImage('player', ...)` with canvas source caused Phaser texture issues
- **Fix**: Changed to `textures.addCanvas('player', canvas)` for player/gopnik aliases

### 5. Missing `Booster` class import in GameScene
- **File**: `gopnik-runner/src/scenes/GameScene.js`
- **Issue**: Used `new Booster(...)` but only imported named `getRandomBoosterType`, not the default class
- **Fix**: Changed import to `import Booster, { getRandomBoosterType } from '../entities/Booster.js'`

### 6. Blurry pixel art visuals
- **File**: `gopnik-runner/src/utils/SpriteGenerator.js`
- **Issue**: Phaser defaults to linear texture filtering, making pixel art sprites blurry when scaled up via `setDisplaySize()`
- **Fix**: Added `texture.setFilter(Phaser.Textures.FilterMode.NEAREST)` in `renderPixels()` for crisp pixel art

### 7. Game speed too fast for screenshot testing
- **File**: `gopnik-runner/src/config.js` (temporarily modified for testing)
- **Note**: Reduced `runSpeedMin/runSpeedMax` from 300/600 to 50/80 to capture gameplay screenshots. Should be reverted to original values for production.

## Game Architecture Notes
- Uses Phaser 3.70 with Arcade Physics
- Procedurally generates pixel-art sprites from string arrays (SpriteGenerator.js)
- Scene flow: IntroScene → GameScene → OutroScene (on death) → restart
- Supabase integration present but disabled when keys missing (graceful degradation)
- Server is simple Express static + SPA fallback on port 5174
