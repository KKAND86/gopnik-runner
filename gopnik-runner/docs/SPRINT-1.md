# Спринт 1 — Core Loop + Pixel-Art

## Цель
Playable прототип runner'а в Telegram Mini App. Пиксельная графика, управление через тап, бесконечный бег.

## Принятые решения

### Стек
- **Phaser 3** (CDN ESM) + vanilla JS — ES modules + importmap, без сборщика.
- **Supabase** — заложен API-слой, активируется в Спринте 3.

### Управление
- **Тап** = обычный прыжок
- **Удержание > 200 мс** = длинный прыжок (выше и дальше)

### Ориентация
- **Ландшафт** 1280×720 логических пикселей
- Масштабирование `Phaser.Scale.FIT`

### Визуал: программный pixel-art
Все спрайты сгенерированы через Phaser Graphics API (16×16 → upscale до 48×64):
- **Игрок** — парень в куртке и джинсах
- **Гопник** — в Adidas и тапочках, злые брови
- **Монетка** — золотой рубль (₽)
- **Бустеры** — кефир, Nokia 3310, мама с метлой

## Что реализовано
- [x] Пиксельный игрок с анимацией наклона при прыжке/падении
- [x] Два типа прыжков (hold detection)
- [x] Пиксельные гопники с forgiveness hitbox (меньше визуала)
- [x] Пиксельные монетки ₽
- [x] Бесконечный спавн препятствий и монет
- [x] Параллакс фона
- [x] UI: дистанция, монетки, Game Over экран
- [x] Очистка объектов за экраном
- [x] Telegram Mini App API (expand, fullscreen, haptic)
- [x] localStorage для рекорда
- [x] Dev-сервер и сборка в `dist/`

## Как запустить
```bash
cd gopnik-runner
npm run dev      # localhost:5173
npm run build    # dist/
```

## Архитектура
- `SpriteGenerator.js` — программная генерация всех спрайтов, нет внешних PNG
- `config.js` — весь баланс в одном месте
- Управление через pointer + keyboard (для десктоп-теста)
- `Player.speed` линейный рост до `runSpeedMax` — сложность автоскейлится
